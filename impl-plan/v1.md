# Component 1: Central Task Coordinator (CTC) - 中央タスク管理機構
これは分散ノードからの要求を処理するステートフルなモジュールです．

## 【データエンティティ定義 (Data Entities)】
Task Entity:
- id: UUID (String)
- capability_level: Enum (1: HIGH_VRAM, 2: FAST_COMPUTE, 3: LOW_RESOURCE)
- state: Enum (AVAILABLE, LOCKED, COMPLETED, FAILED)
- payload: String (Markdown/YAML形式のタスク詳細)
- lock_metadata: Object (ロック取得者のID, 取得タイムスタンプ)
- history: Array of Objects (過去の失敗理由等のログ)

## 【状態遷移ルール (State Machine)】
AVAILABLE のタスクのみが取得対象となる．
取得されると LOCKED に遷移する．この遷移処理は**アトミック(Atomic)**に行われなければならない．同時に2つのリクエストが来た場合，片方は必ずブロックまたは拒否されること(Mutex/Lock機構の必須要件)．
タイムアウト，または明示的な返却リクエストで AVAILABLE に戻る．

## 【インターフェース・コントラクト (Interface Contracts)】
ネットワークプロトコル(REST, gRPC, WebSocket等)は問わないが，以下のシグネチャを持つ関数/エンドポイントを実装すること．

- AcquireTask(requester_id: String, capability: Enum) -> Task | Error(Conflict)
事前条件: 該当の capability 以下の AVAILABLE なタスクが存在すること．
処理: 最も優先度の高いタスクを検索し，アトミックに状態を LOCKED に変更して返す．

- YieldTask(requester_id: String, task_id: UUID, reason: String) -> Success | Error
事前条件: task_id が requester_id によってロックされていること．
処理: history に reason を追記し，状態を AVAILABLE に戻す．

# Component 2: Semantic Environment Translator - 意味的翻訳モジュール
Minecraftクライアント(エージェント)内で動作し，生の環境データをLLMが理解できるテキストに変換する純粋関数(Pure Functions)の集合です．

## 【処理ロジック要件】
- Function: TranslateSpatialData(self_position: Vector3, target_positions: List[Vector3], threshold_distance: Float) -> String
要件: 自身とターゲット間のユークリッド距離を計算する．
制約: 座標の生数値を出力に含めてはならない．距離の数値と，相対的な方角(North, South等)のみを含むフォーマットされた文字列を返すこと．threshold_distance を超える場合は"相互作用不可"のステータスを付与すること．

- Function: FilterInventoryByContext(inventory: Dict, task_keywords: List[String]) -> String
要件: インベントリ全体を走査せず，task_keywords にマッチするアイテムの数量のみを抽出し，サマリーテキストとして出力すること．

# Component 3: LLM Gateway & Constrained Decoder - 出力制約機構
推論サーバー(vLLM等)と通信し，出力フォーマットを強制するプロキシ層です．

## 【処理ロジック要件】
- Function: GenerateAction(prompt: String, schema_definition: Object) -> ParsedObject | Error
要件: 推論API(vLLM等のエンドポイント)に対し，プロンプトと共に"期待する出力スキーマ(JSON Schema等)"をペイロードに含めてリクエストを送信する．
フォールバック要件: パースエラーが発生した場合，システムをクラッシュさせず，エラーメッセージをプロンプトに末尾追加して再帰的に最大N回までリトライするループ機構を持つこと．
